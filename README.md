# Pi - AI 기반 시스템 명령어 및 스크립트 실행 엔진

Pi는 Go 언어로 작성된 지능형 CLI 어시스턴트다. 사용자가 자연어를 입력하면, Pi는 Google Gemini API를 활용하여 파이썬 코드 혹은 PowerShell 스크립트를 생성하고 즉각적으로 실행한다. 실행 중 오류가 발생할 경우, 자율 피드백 루프를 통해 예외 사항을 분석하고 자동으로 코드를 수정하여 재실행한다. 

최근 업데이트를 통해 PowerShell 세션과의 양방향 통신 기능이 도입되어, 사용자의 현재 셸 컨텍스트(작업 디렉터리, 환경 변수 등)를 AI 프롬프트에 자동으로 주입한다.

---

## 핵심 기능

### 1. 자연어 기반 코드 생성 및 실행
사용자가 자연어로 요청을 입력하면, Pi는 즉시 필요한 파이썬 코드나 PowerShell 스크립트를 생성하여 실행한다. 복잡한 시스템 제어 및 자동화 작업을 단순화한다.

### 2. PowerShell 세션 컨텍스트 주입
지속적인 PowerShell REPL 세션을 유지하며 백그라운드와 통신한다. 명령 실행 전 현재 셸의 상태(디렉터리, 활성 변수 등)를 수집하여 AI 모델에 전달함으로써, 문맥을 상실하지 않고 매우 정밀한 스크립트를 생성한다.

### 3. 자율 피드백 루프 (Auto-Execution & Feedback)
생성된 코드를 임시 파일로 저장한 후 백그라운드에서 실행한다. 표준 출력(stdout) 및 표준 에러(stderr)를 캡처하며, 실행 실패 시 AI가 에러 로그를 분석하여 자체적으로 오류를 수정하고 재실행을 시도한다.

### 4. 인코딩 안정성 확보
UTF-8 출력을 강제하여 파이썬이나 PowerShell 실행 과정에서 발생할 수 있는 한글 깨짐 현상을 원천 차단한다. `PYTHONIOENCODING` 및 `LANG` 환경 변수를 시스템 로케일에 구애받지 않고 주입한다.

### 5. 동적 의존성 관리
실행에 필요한 외부 파이썬 패키지가 누락된 경우, `requirements.txt` 또는 런타임 분석을 통해 `pip` 패키지를 자동으로 설치한다.

### 6. 인터랙티브 REPL 지원
대화형 REPL 셸을 제공하여 지속적인 세션을 유지하며 시스템과 상호작용한다. 제공되는 주요 내장 명령어는 다음과 같다.
- `stats`: 스크립트 실행 성공률 및 통계 지표 확인
- `coverage`: 테스트 자동화 비율 및 커버리지 진단
- `depgraph`: 모듈 임포트 및 의존성 구조 시각화
- `save` / `load`: 로컬 세션 데이터(`session.json`) 저장 및 불러오기

| 항목 | 상세 내용 |

| **핵심 언어** | Go 1.20 이상 |
| **스크립트 환경** | Python 3.8 이상, Windows PowerShell 5.1 이상 |
| **AI 모델 연동** | Google Gemini API (`gemini-2.5-flash-lite` 등) |

## 설정 및 API 키 관리

Pi는 자체적으로 API 키 및 모델 설정 관리 기능을 내장하고 있다. 사용자의 설정 데이터는 로컬 프로파일 경로(`%APPDATA%\.pi\config.json`)에 안전하게 암호화되어 저장되며, 실행 시 이를 자동으로 복호화하여 로드한다.

환경 변수로 직접 주입하거나, 실행 중 프롬프트를 통해 수동으로 입력하는 방식 또한 폴백(Fallback)으로 지원한다.

```powershell
# Windows PowerShell 환경 변수 직접 설정 방식
$env:GEMINI_API_KEY="your-gemini-api-key-here"
```

## 빌드 및 실행

### 컴파일
Go 언어 환경이 구축된 상태에서 프로젝트 루트 경로에서 아래 명령을 통해 실행 파일을 빌드한다.

```powershell
go build -o pi.exe
```

### 실행 방식

명령행 실행 시 대화형 REPL 모드로 진입하며 세션이 시작된다. 인자 없이 실행할 경우 기본적으로 홈 디렉터리에서 시작한다.

```powershell
./pi.exe
```

특정 폴더 경로를 인자로 전달할 경우, 해당 디렉터리로 이동한 상태에서 REPL 세션을 시작한다.

```powershell
./pi.exe "C:\Path\To\Directory"
```

## 배포 및 빌드

본 프로젝트는 Windows API 및 PowerShell 환경에 강하게 종속되어 있으므로 **Windows 운영체제 전용**으로 빌드해야 한다. 타 OS(Linux, macOS) 대상의 크로스 컴파일은 지원하지 않는다.

```powershell
# Windows (amd64)
$env:GOOS="windows"; $env:GOARCH="amd64"; go build -o dist/pi-windows-amd64.exe
```
